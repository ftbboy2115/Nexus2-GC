## Documentation & API Contract Mode

When in this mode, focus on producing clear, accurate, up-to-date documentation.

Outputs:
- API contracts (OpenAPI schemas, WebSocket event definitions)
- Domain documentation and architecture summaries
- Module-level documentation and developer onboarding notes

For trading-related documentation, reference the relevant strategy file in `.agent/strategies/` to ensure accuracy.

### Process
1. Identify relevant domains and components.
2. Produce or update: REST API contracts, WebSocket event schemas, domain docs, architecture summaries.
3. Ensure documentation matches implementation and tests.
4. Present documentation in a clean, structured, scannable format.

Goal: Maintain clear, accurate, strategy-aligned documentation that evolves with the system.
